# SEO 가이드/FAQ 페이지 설계

## 목표
"영문주소 변환기", "영문주소 변환" 키워드로 구글 상위 노출.
정적 서버 컴포넌트 기반 가이드/FAQ 페이지를 추가해 구글 크롤러가 읽을 수 있는 텍스트 콘텐츠를 확보한다.

---

## 추가할 페이지

### 1. `/address/guide` — 영문주소 변환기 사용 가이드

**파일**: `app/address/guide/page.tsx` (서버 컴포넌트)
**메타 타이틀**: `영문주소 변환기 사용 가이드 — 해외배송·직구 영문주소 만들기`
**메타 디스크립션**: `아마존, 알리익스프레스, 몰테일 등 해외배송·직구 시 영문주소 입력 방법을 단계별로 설명합니다. 도구함 영문주소 변환기로 3단계 만에 완성하세요.`

**페이지 구성**:
1. H1: 영문주소 변환기 사용 가이드
2. 영문주소가 필요한 상황 (아마존, 알리익스프레스, 몰테일, 배대지)
3. 영문주소 구성 요소 설명
   - Address Line 1: 건물번호 + 도로명
   - Address Line 2: 동/호수/건물명 (상세주소)
   - City: 구(區) 단위
   - State / Province: 시/도 단위
   - ZIP Code: 우편번호
   - Country: South Korea
4. 3단계 사용법
   - Step 1: 도로명 주소 검색 (카카오 주소 검색)
   - Step 2: 상세주소 입력 (동/호수)
   - Step 3: 항목별 복사해서 사이트에 붙여넣기
5. 사이트별 입력 가이드 요약 (아마존 / 알리익스프레스 / 몰테일)
6. CTA: "지금 바로 변환해보기" → `/address` 링크

---

### 2. `/address/faq` — 자주 묻는 질문

**파일**: `app/address/faq/page.tsx` (서버 컴포넌트)
**메타 타이틀**: `영문주소 변환 자주 묻는 질문 (FAQ) — 도구함`
**메타 디스크립션**: `영문주소 변환기 사용법, 아마존·알리익스프레스·몰테일 주소 입력 방법 등 자주 묻는 질문에 답합니다.`

**FAQ 항목** (FAQ JSON-LD 구조화 데이터 포함 → 구글 리치 스니펫 노출 목표):
1. 영문주소 변환기가 무엇인가요?
2. 무료로 사용할 수 있나요?
3. Address Line 1과 Address Line 2에 각각 뭘 입력하나요?
4. City와 State에 뭘 입력해야 하나요?
5. 아마존에서 한국 주소를 영문으로 입력하는 방법은?
6. 알리익스프레스 영문주소 입력 방법은?
7. 몰테일 영문주소 입력 방법은?
8. 변환 결과가 정확한가요?
9. 상세주소(동/호수)는 어떻게 입력하나요?
10. 아파트 동/호수를 영문으로 어떻게 쓰나요?

## 기존 페이지 개선

### `app/address/page.tsx` 하단 콘텐츠 추가
현재 `'use client'`라 구글 봇이 읽을 텍스트가 없음.
도구 UI(`AddressPageClient`) 아래에 서버 렌더링 텍스트 블록 추가:
- "가이드 보기" / "자주 묻는 질문" 링크 카드
- 간략한 서비스 소개 텍스트 (크롤러용)

**구현 방법**: 페이지를 서버/클라이언트로 분리
- `app/address/page.tsx` → 서버 컴포넌트 (레이아웃 + 하단 콘텐츠)
- `components/address/AddressPageClient.tsx` → 기존 클라이언트 로직 이동

## 기술 요구사항

- 모든 새 페이지는 **서버 컴포넌트** (구글 크롤링 가능)
- FAQ 페이지에 **FAQ JSON-LD** 구조화 데이터 추가
- 각 페이지 개별 `metadata` export (타이틀/디스크립션/canonical)
- 기존 클라이언트 로직은 그대로 유지 (리팩토링 최소화)
