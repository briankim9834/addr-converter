# 영문주소 변환기 서비스 — 설계 문서

**작성일:** 2026-04-06
**프로젝트:** engaddr.kr (가칭)
**목적:** 한글 주소를 해외배송·직구에 필요한 항목별 영문 주소로 변환해주는 웹 유틸 서비스. 추후 멀티 유틸 플랫폼으로 확장 예정.

---

## 1. 서비스 개요

### 타겟 사용자
20~30대 직장인. 해외직구·해외배송 시 영문 주소 입력이 번거로운 사람.

### 핵심 Pain Point
- 영문 주소 변환 후 각 사이트의 입력 필드(Address Line 1/2, City, State, ZIP 등)에 맞게 직접 나눠서 붙여넣기 해야 함
- 어느 값이 어느 필드에 들어가는지 매번 헷갈림
- 상세주소(동/호수/건물명)를 영문으로 어떻게 표기해야 할지 모름

### 해결 방법
1. 카카오 주소 자동완성으로 정확한 주소 검색
2. JUSO API로 공식 영문 주소 변환 (항목별 분리)
3. Papago API로 상세주소 자동 번역 (편집 가능)
4. 각 항목별 복사 버튼 + 전체 복사 버튼 제공
5. 사이트별(Amazon, AliExpress, 몰테일) 입력 가이드 제공

## 3. 사이트 구조 (멀티 유틸 플랫폼)

```
/ (홈)                → 도구 카드 그리드
/address              → 영문주소 변환기 (v1 출시)
/[다음 도구]           → 추후 추가
```

### 홈 페이지 (`/`)
- 도구 카드 그리드 (2열)
- 각 카드: 아이콘, 도구명, 한 줄 설명, "바로 사용" 버튼
- 빈 카드는 "도구 추가 예정" placeholder로 표시
- SEO: 타이틀 "직장인을 위한 실용 유틸 도구 모음"

### 영문주소 변환기 페이지 (`/address`)
- SEO 타이틀: "영문주소 변환기 — 한글 주소를 영문으로 즉시 변환"
- SEO 키워드: 주소영문변환기, 주소 영문변환, 영문주소변환, 해외배송 영문주소

## 4. 영문주소 변환기 — 기능 상세

### 4-1. 사용자 흐름 (3단계 + 결과)

**STEP 1 — 초기 화면**
- 빈 검색창 (포커스 상태)
- 간단한 사용 안내 (3줄)

**STEP 2 — 주소 입력**
- 도로명/지번 입력 시 카카오 자동완성 드롭다운 표시
- 주소 선택 시 STEP 3으로 이동

**STEP 3 — 상세주소 + 변환**
- 선택된 도로명 주소 확인 표시 (초록색, "변경" 링크 포함)
- 상세주소 입력 필드 (선택사항): 동/호수/건물명 한글 입력
- "영문 주소로 변환하기" 버튼 → 클릭 시 결과 표시

**결과 화면**
- 상단에 현재 주소 요약 카드 + ✏️ 수정 / ↺ 초기화 버튼
- 항목별 결과 카드 (세로 단일 컬럼)
- 하단에 전체 복사 버튼

### 4-2. 변환 결과 카드 구성 (세로 순서)

| 순서 | 필드명 표기 | 비고 |
|---|---|---|
| 1 | Street Address 1 / Address Line 1 | JUSO API 영문 변환값 |
| 2 | Street Address 2 / Address Line 2 | Papago 번역값, 편집 가능 필드 + ⚠️ 안내 문구 |
| 3 | City | JUSO API 영문 변환값 |
| 4 | State / Province / Region | JUSO API 영문 변환값 |
| 5 | Zip Code / Postal Code | 우편번호 |
| 6 | Country | "South Korea" 고정값 |
| 7 | 전체 복사 | 전체 주소 한 줄 조합, 강조 버튼 |

**Address Line 2 특이사항:**
- 상세주소를 Papago API로 번역한 결과를 편집 가능한 텍스트 필드로 표시
- ⚠️ 안내 문구: "AI 번역 결과예요. 클릭해서 수정 후 복사하세요."
- 상세주소 미입력 시 Address Line 2 카드는 숨김 처리 (표시 안 함)

**JUSO API 필드 매핑:**
JUSO API의 개별 필드를 사용해 항목별로 구성 (engAddr 단일 문자열 파싱 방식 사용 안 함):
- Address Line 1: `{lnbrMnnm}-{lnbrSlno} {rn}, {sggNm}` (건물번호 + 도로명 + 구)
- City: `siNm` (시/도)
- State / Province: `siNm` (한국은 City와 동일하게 사용)
- Zip Code: `zipNo`
- Country: "South Korea" (고정값)

**에러 처리:**
- JUSO API 응답 실패 시: "주소 변환에 실패했어요. 다시 시도해주세요." 안내
- Papago API 응답 실패 시: 상세주소 필드에 원본 한글 그대로 표시 + 수동 입력 유도

### 4-3. 수정 / 초기화 동선

**✏️ 수정 버튼**
- 결과 위에 수정 패널이 인라인으로 펼쳐짐
- 도로명 주소: "재검색" 링크로 카카오 자동완성 재활성화
- 상세주소: 직접 편집 가능
- "다시 변환" 클릭 시 결과 업데이트, "취소" 클릭 시 패널 닫힘
- 수정 중에는 기존 결과 카드 흐리게(opacity) 표시

**↺ 초기화 버튼**
- 모든 입력값과 결과를 초기화
- STEP 1 첫 화면으로 완전히 리셋

### 4-4. 사이트별 입력 가이드

탭 형태로 전환:
- 🛒 Amazon
- 🛍️ AliExpress
- 📦 몰테일
- (추후 탭 추가 가능한 구조)

각 탭: 해당 사이트의 실제 입력 필드명 기준으로 값 매핑 표시 (복사 버튼 없이 참고용)

## 5. 디자인 시스템

### 색상
| 역할 | 색상 |
|---|---|
| Primary (메인) | Indigo `#6366F1` |
| Primary Dark | Purple `#7C3AED` |
| Gradient | `linear-gradient(135deg, #6366F1, #8B5CF6)` |
| Background | `#F5F3FF` (연보라) / `#F9FAFB` (연회색) |
| 텍스트 주요 | `#1E1B4B` (다크 인디고) |
| 텍스트 보조 | `#6B7280` |

| 삭제/초기화 | Red `#EF4444` |

### 레이아웃
- 도구 페이지 최대 폭: `max-width: 560px`, 중앙 정렬
- 모바일 우선 반응형
- 카드 border-radius: `10px`
- 버튼 border-radius: `6px`

### 타이포그래피
- 제목: `font-weight: 800`
- 필드 레이블: `font-size: 9px`, 대문자, `letter-spacing: 0.5px`
- 필드 값: `font-size: 11~12px`, `font-weight: 500`

- 한국어/영어 양쪽 메타 태그
